# Configuration

AGIAAS requires several environment variables to function correctly. These are managed via a central `.env` file in the root of the monorepo. Use `pnpm setup` to generate it interactively, or create it manually.

## Required Variables

### Security & Encryption

These keys are generated automatically by `pnpm setup`.

| Variable | Description |
| :--- | :--- |
| `DB_ENCRYPTION_KEY` | 32-character key used to encrypt repository tokens in the database (AES-256-GCM). |
| `BETTER_AUTH_SECRET` | 32+ character key used for session management and dashboard security. |

### AI Provider

AGIAAS supports **OpenRouter** and **Nous Research** for LLM inference. At least one key is required.

| Variable | Description |
| :--- | :--- |
| `OPENROUTER_API_KEY` | Your OpenRouter API key (optional if using Nous). |
| `NOUS_API_KEY` | Your Nous Research API key (starts with `sk-2yd`). |
| `HERMES_AGENT_URL` | URL where the Python Agent is reachable (default: `http://localhost:8678`). |

### GitHub Integration

| Variable | Description | Example |
| :--- | :--- | :--- |
| `GITHUB_TOKEN` | Personal Access Token with `repo` and `workflow` scopes. | `ghp_...` |
| `GITHUB_USERNAME` | Your GitHub account username. | `mehmetkr-31` |

### Messaging (Telegram)

| Variable | Description | Example |
| :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | Token obtained from [@BotFather](https://t.me/BotFather). | `123456:ABC...` |
| `TELEGRAM_CHAT_ID` | Your personal chat ID to receive alerts. | `987654321` |

### Ports

| Variable | Default | Description |
| :--- | :--- | :--- |
| `PORT` | `3678` | Dashboard web server port. |
| `WEBHOOK_PORT` | `8678` | Webhook receiver / Agent API port. |

### Optional

| Variable | Description |
| :--- | :--- |
| `AI_BUDGET_LIMIT` | Daily AI spending threshold (USD). Triggers Telegram alert when exceeded. |

## Security Best Practices

- **Never commit your `.env` file**: It is automatically ignored by `.gitignore`.
- **Rotatable Tokens**: Use tokens with short expiration dates where possible.
- **Scoped Tokens**: Ensure your `GITHUB_TOKEN` has the minimum necessary permissions (`repo`, `workflow`).

## Using Secrets in Production

When deploying to Cloudflare Workers/Pages, set variables as **Secrets** in the Cloudflare Dashboard:

```bash
npx wrangler secret put GITHUB_TOKEN
```

> [!TIP]
> The `@agiaas/env` package validates all environment variables at startup using Zod schemas via `@t3-oss/env-core`. If a required variable is missing, you'll see a clear error message immediately.
